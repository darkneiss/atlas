# Architecture Decision Records (ADRs)

This directory contains Atlas Architecture Decision Records.

## Purpose

ADRs capture significant architectural decisions so the team can understand:

- why a decision was needed,
- what was chosen,
- and what trade-offs were accepted.

## Conventions

- File naming: `NNNN-short-title.md` (for example `0001-monorepo.md`).
- Status starts as `Accepted` for active project baselines.
- New decisions should reference and remain consistent with earlier ADRs unless explicitly superseded.
